# Helmetsan Quick Context

**Stack**: WordPress (PHP 8.1+), Custom Plugin + Theme, JSON data layer, GitHub sync.

## Key Paths

| Area | Path |
|------|------|
| Plugin bootstrap | `helmetsan-core/includes/Core/Plugin.php` |
| AI modules | `helmetsan-core/includes/AI/` (AiService, FillMissing, Providers) |
| CLI commands | `helmetsan-core/includes/CLI/Commands.php` |
| Ingestion | `helmetsan-core/includes/Ingestion/IngestionService.php` |
| Sync | `helmetsan-core/includes/Sync/SyncService.php` |
| SEO | `helmetsan-core/includes/Seo/` |
| Theme templates | `helmetsan-theme/` (root PHP, template-parts/, assets/) |
| JSON catalogs | `data/` (helmets, brands, accessories) |
| Schemas | `data/schemas/` |
| Scripts | `scripts/` |

## Common Entry Points

- **AI enrichment**: `FillMissingService.php` → calls `AiService` → routes to provider
- **CLI**: All commands in `Commands.php` — use grep/line-range, file is large
- **Ingestion**: `IngestionService.php` — JSON → WordPress posts/meta
- **Sync**: `SyncService.php` + `Data/SyncManager.php` — GitHub pull/push

## Rules (Summary)

- Never modify `wp-admin/` or `wp-includes/`
- Max 3 files / 200 lines per change
- No hardcoded API keys — use `getenv()` or `get_option()`
- Edit existing services, don't create new subsystems
- Full governance: `.agent/workflows/ai-optimizations.md`
- Full architecture: `docs/architecture-map.md`
